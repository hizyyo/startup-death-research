# The Anatomy of Startup Death

## Project One-Pager

### What It Is

A structured public research dataset and taxonomy of **100 early-stage startup failures**, built from founder postmortems, shutdown announcements, interviews, and public evidence.

### Dataset Snapshot

| Metric | Value |
|---|---|
| Cases | 100 |
| Primary-source cases | 72 |
| High-confidence cases | 69 |
| Schema validation errors | 0 |
| Failure cause categories | 19 |

### Research Question

What patterns emerge when early-stage startup failures are classified using structured public postmortem evidence?

### Methodology

Each case is coded using a consistent schema with 24 fields. Sources are graded as primary (founder postmortems, interviews, official announcements) or secondary (detailed reporting with founder quotes).

### Top Primary Failure Causes

| Cause | Count |
|---|---|
| product-market fit failure | 14 |
| no urgent pain | 11 |
| business model mismatch | 9 |
| operational complexity | 8 |
| distribution failure | 7 |
| regulatory or legal pressure | 6 |

### Key Finding

Most startup deaths do not come from one dramatic event. They come from a **mismatch** between product, urgent customer need, monetization, operations, platform dependency, and founder motivation.

### Strongest Evidence

- 72% of cases use primary source material
- 69% are high-confidence classifications
- founder narratives, not speculation

### Known Limitations

- only failures with public evidence
- North America overrepresented (71%)
- descriptive, not predictive
- founder accounts may be self-serving

### Why It Matters

Startup failure is usually discussed through anecdotes. This project makes failure patterns structured, transparent, and analyzable.
