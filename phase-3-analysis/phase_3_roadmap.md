# Phase 3 Roadmap

# The Anatomy of Startup Death

## Phase 3 Goal

Turn the 100-case MVP dataset into a credible public research project.

Phase 3 is not about adding more cases. It is about making the existing work understandable, auditable, and impressive to an outside reader.

## Quality Bar

The project should feel like independent research, not content curation.

Every public artifact should show:

- clear research question
- transparent methodology
- structured dataset design
- honest limitations
- reproducible summary statistics
- evidence-based claims
- restraint around overclaiming

## Required Outputs

## 1. Public README

Purpose:

Make the project understandable in 60 seconds.

Must include:

- project description
- why it matters
- dataset snapshot
- methodology overview
- taxonomy overview
- early findings
- limitations
- project structure
- status and next steps

## 2. Analysis Tables

Purpose:

Make the dataset explorable without requiring code.

Must include:

- failure cause distribution
- source quality distribution
- confidence distribution
- stage distribution
- region distribution
- category distribution
- category-by-primary-cause table
- stage-by-primary-cause table

## 3. Research Memo

Purpose:

Show original thinking from the dataset.

Must include:

- claims grounded in dataset statistics
- examples from cases
- limitations and caution
- follow-up research questions

## 4. Admissions Summary

Purpose:

Help explain the project in applications, interviews, and activity descriptions.

Must include:

- concise one-paragraph version
- longer project description
- intellectual motivation
- skills demonstrated
- why it is unusual for a high-school applicant

## 5. Landing Page Copy

Purpose:

Prepare the project for a future website.

Must include:

- hero section
- dataset stats
- research framing
- call to explore dataset
- limitations statement

## Phase 3 Definition Of Done

Phase 3 is complete when:

- the README can stand alone on GitHub
- the dataset has analysis tables
- the project has an honest limitations page
- the project has at least one research memo
- an admissions officer can understand why this project is serious without needing extra explanation
- validation still passes after all changes
