# The Anatomy of Startup Death

## Hero

Startup failure is usually told as a story.

This project turns those stories into structured research data.

The Anatomy of Startup Death is a public dataset and taxonomy of 100 early-stage startup failures, built from founder postmortems, shutdown announcements, interviews, and public evidence.

## Research Framing

Most startup failure advice is anecdotal.

Founders write postmortems. Investors write essays. Journalists report shutdowns. But these accounts are rarely organized into a structured dataset.

This project asks a simple question:

What patterns emerge when startup failures are classified using a consistent taxonomy and public evidence?

## What The Dataset Includes

Each case includes:

- startup description
- source URL
- source type and quality
- country and region
- startup category
- customer type
- business model
- stage at failure
- funding where available
- stated failure cause
- primary classified failure cause
- secondary failure causes
- evidence summary
- confidence score

## Early Finding

The first 100 cases suggest that startup death is usually not one dramatic event.

It is more often a mismatch between product, urgent customer need, monetization, operations, platform dependence, regulation, and founder motivation.

## Failure Categories

The taxonomy includes categories such as:

- product-market fit failure
- no urgent pain
- business model mismatch
- distribution failure
- marketplace liquidity failure
- operational complexity
- technical complexity
- platform risk
- regulatory or legal pressure
- trust and safety failure
- founder conflict

## Limitations Statement

This dataset does not claim to represent all startup failures.

It studies failures with public evidence, which creates bias toward startups whose founders wrote postmortems, gave interviews, or received media coverage.

The project is descriptive, not predictive.

## Footer Description

The Anatomy of Startup Death is an independent research project about entrepreneurship, data, and failure. It was built to make startup postmortems more structured, comparable, and useful for learning.
