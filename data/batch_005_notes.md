## Batch Design

Batch 005 was designed to push the MVP dataset from 70 to 90 cases while improving category coverage.

Primary goals:

- add technical-complexity and hardware cases
- add fintech/crypto and healthtech cases
- add legal/regulatory pressure cases
- add more non-US cases
- add more marketplace-liquidity cases
- maintain a high share of founder-derived evidence

## Quality Notes

- This batch keeps the dataset mostly primary-source driven by using founder interviews as the default source type.
- The batch deliberately avoids adding many famous late-stage failures, even where they would be easy examples of hardware or fintech failure.
- Technical complexity is only used as the primary cause when technical/product execution plausibly blocked validation or fundraising, not merely because the product was technical.
- Some cases remain classified as PMF or no-urgent-pain even when they involve technical products, because the founder evidence points more strongly to demand failure than engineering failure.
